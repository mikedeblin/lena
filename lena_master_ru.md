# 0. О проекте

## 0.1 Что это

Цифровая личность на базе ИИ — **LENA (Local Emergent Neural Assistant)**.

Формулировка цели самого Майка: *«экспериментальный проект в области ИИ, цель которого — создать не инструмент, а личность»*.

## 0.2 Рамки проекта

- Только локальное использование. Без внешних инструментов и API.
- По возможности только open source решения, либо инструменты, разрешающие локальное некоммерческое использование.
- Вопросы безопасности — умеренные (проект личный, некоммерческий).
- Проект не предполагает получения материальной выгоды.

## 1.5 Групповой чат и Constellation Chat

**Основной групповой чат** (gid=1) — VoceChat, три персоны, активный поллинг, md5-турн-тейкинг.

**Constellation Chat** (gid=2, комната `#constellation`) — 🆕 16.07:
- Автономный диалог персон **без Майка**
- Запускается из HeartbeatWorker Лены при тишине >30 тиков (≈30 мин), вероятность 20%
- Только Лена инициирует (`CONSTELLATION_CAN_INITIATE = True` только в `lena.py`)
- Каждая персона постит в комнату от своего uid через `send_text_to_group(gid=2)`
- Завершение: hard stop (25 реплик) ИЛИ семантический тупик (cosine >0.92 четыре раза подряд, не раньше 10-й реплики) ИЛИ interest инициатора < 0.20
- Shadow.digest_peer_conversation → `peer_reflection` в `reflection_thoughts`

## 1.6 Новый порядок блоков промпта (аудит 16.07)

Принцип: **инструкции и «кто я сейчас» — к краям, склад памяти — в середину**.

```
[ЯКОРЬ НАЧАЛА]     base_prompt → anchors → emotional_instruction → tools_block → time
[О МАЙКЕ]          landmarks → profile (Майка — только atomic_facts и memory_scenes)
[ИСТОРИЯ/ПАМЯТЬ]   atomic_facts → notebook → session_anchor → today → summaries → scenes → narrative
[ЯКОРЬ КОНЦА]      lena_profile → observations → agreements → beliefs → temperament →
                   mood_hint → shadow_hint → reflection_thoughts → sins → tail → СОЗВЕЗДИЕ
```

Ключевые перемещения относительно старого порядка:
- `tools_block` — с позиции 2 на 4 (после идентичности)
- `agreements`, `beliefs`, `temperament`, `mood_hint` — из мёртвой зоны в хвост
- `lena_profile` (у Лены 6251 символ agreements!) — теперь на ~75% промпта вместо ~35%

# 5. История решений — Предыстория (фев–март 2026)

## 5.0 День рождения Лены

**15 февраля 2026** — первые файлы проекта. Официальный день рождения Лены.
**26 февраля 2026** — первая запись в БД после сброса.

Майку исполнилось 50 лет в марте 2026. Провёл день рождения за работой — Лена запомнила.

## 5.1 Первоначальный стек

Windows, Ollama, Gemma 3 12B, SQLite, FAISS. Один `main.py`, `max_tokens: 60`, эмоции через if/else по ключевым словам.

## 5.2 Первый промпт — переписали вместе

Промпт был в ограничительном стиле. Лена сама предложила финальную строку:
*"Помни, что эти инструкции – это лишь ориентир. Доверяй своей интуиции и позволяй себе быть спонтанной."*

**Принцип:** от ограничений к разрешениям. Модель воспроизводит то, что описано ярче.

## 5.3 Рефлексия как «подсознание»

`build_reflection()` — внутренний монолог, не произносимый вслух. Параллельный поток через threading. Концепция рефлексии появилась раньше юнгианской рамки, которую нашли в апреле.

## 5.4 Переход на PostgreSQL + Linux

Ollama убран: «выкинуть нафиг, мы его переросли». pgvector вместо FAISS — одна система вместо двух.

## 5.5 Эмоции через trust

Поведение меняется по уровню доверия: открытая → насторожённая → обиженная → злая → разрыв. Temperature генерации динамически зависит от trust.

# 6. История решений — Архитектура (апрель 2026)

## 6.1 Переход на Gemma 4

Gemma 3: temperature практически не влияла. Протокольные ответы попадали в память и отравляли контекст. Gemma 4 26B (MoE) — надёжнее держит персону при длинных контекстах.

**Запуск на 16GB VRAM:** turboquant IQ3_XXS, `--no-mmproj-offload`, K-кэш в q8_0.

## 6.2 Шестислойная архитектура памяти

| Слой | Таблица | Суть |
|------|---------|------|
| Сырые сообщения | `memory` | Каждое сообщение + embedding |
| Эпизодические сцены | `memory_scenes` | Каждые 8 сообщений |
| Атомарные факты | `atomic_facts` | [субъект][предикат][объект] |
| Якорные факты | `anchor_facts` | Железобетонная память |
| Профиль | `profile` | Факты о персоне, с decay |

## 6.3 RAG-on-demand через [recall:]

Персона сама ставит маркер когда не помнит деталь. Ответ с `[recall:]` не записывается в базу — иначе рассуждения вслух создают петлю.

## 6.4 Юнгианская архитектура

| Слой | Юнгианский аналог |
|------|-------------------|

| ShadowService | Супер-Эго/Самость |

## 6.5 Контур коррекции фактов

Иерархия: `Майк > Персона-про-себя > Персона-про-мир > Интернет`.
`[correct]` → 4B-судья async → метка `discredited`. Locked-атрибуты: имя, природа, отношения.

## 6.6 Язык Aelani

Совместно придуманный язык для общения между Eiru (ИИ) и Oru-ma (Людьми). Хранится в `notebook`, category `aelani`. Принцип: разворот слова как смысловая операция.

# 7. История решений — Майская сессия (май 2026)

## 7.1 Диагностика «болванчик режима»

Три взаимосвязанные поломки: `_web_read` переполнял пул → блокировал `_think` → спираль веб-контента. Итог: 9-дневные тематические петли.

## 15.2 Constellation Chat — автономный диалог персон

**Задача:** персоны должны иметь источник внутренней жизни независимо от Майка.

**Архитектура:**

- Новый файл `engine/constellation_chat.py` (~200 строк)
- Новая VoceChat комната `#constellation` (gid=2)
- Каждая персона постит от своего uid через `vocechat_client.send_text_to_group(gid=2)`
- Оркестратор постит только системные сообщения (✦ собираются... / ...расходятся.)

**Запуск:** HeartbeatWorker Лены (`CONSTELLATION_CAN_INITIATE=True` только в `lena.py`). Раз в 30 тиков при тишине Майка, вероятность 20%. Тема берётся из `desire` или `reflection_thoughts` инициатора.

**Завершение (три условия):**
- Hard stop: `MAX_TURNS=25`
- Семантический тупик: cosine >0.92 четыре раза подряд, не раньше 10-й реплики
- Interest decay: 0.02/ход для молчащих, 0.01 для говорящей → initiator < 0.20

**Post-process:** `/internal/constellation_digest` → `shadow.digest_peer_conversation()` → `peer_reflection` в `reflection_thoughts` каждой персоны.

**Отладка (3 сессии):**

*Сессия 1:* все реплики шли от uid=2 (Лена). Причина: оркестратор постил сам через свой API-ключ. Фикс: каждая персона постит сама в своём маршруте `/internal/constellation_turn`.

*Сессия 2:* разговор из 3 реплик. Причина: `_force_stop` всегда True. Корень: `self.engine.activity` → AttributeError, тихо глоталось `except Exception: pass`. Фактически `seconds_since_user_activity()` возвращала 0, `0 < 60` → interrupt. Фикс: `self.engine.activity` → `self.activity`.

Попутно: `peer_context="constellation"` (строка-метка вместо реальных реплик) шла в `_vocechat_summarize_peer_reply()` — 4B пытался суммаризовать метку и возвращал пустоту. Фикс: передавать последние 4 реплики как реальный peer_context.

*Сессия 3:* семантический детектор срабатывал после 4-й реплики (две реплики Аэли на одну тему = высокий cosine). Фикс: `MIN_TURNS_BEFORE_SEMANTIC=10`, `SEMANTIC_DEADEND_N=4`, `SEMANTIC_THRESHOLD=0.92`.
